소개

filePort는 파일 첨부 UI를 빠르게 구현할 수 있는 경량 라이브러리입니다. Dropzone.js v5를 내장하고 있어 별도 설치 없이 드래그&드롭을 바로 사용할 수 있으며, 문서 종류 분류·업로드 상태 추적·진행률 표시 등 실무에 필요한 기능을 기본 제공합니다.

📦

Dropzone 내장

Dropzone.js v5.9.3 번들 포함. 별도 설치·CDN 링크 불필요.

🟦

TypeScript

완전한 타입 정의 제공. IDE 자동완성과 타입 안전성을 보장합니다.

⚡

ESM + IIFE 이중 지원

모던 번들러(Vite, Webpack)와 순수 HTML <script> 태그 모두 지원.

🗂️

문서 종류 분류

파일별 드롭다운으로 문서 종류 선택. 백엔드 연동 데이터 포함.

📊

상태 추적 & 진행률

pending · uploading · success · error 상태와 실시간 퍼센트 바.

🖨️

스캐너 API 등 외부 경로로 추가한 파일도 동일한 UI로 관리.

빠른 시작

가장 간단한 사용 예시입니다. HTML에 구조를 추가하고 두 줄로 업로더를 초기화합니다.

const uploader = FilePort.createUploader({
    uploadUrl: '/api/upload',

    onSubmit: function({ files, extra, done }) {
        fetch('/api/contents/save', {
            method: 'POST',
            headers: { 'Content-Type': 'application/json' },
            body: JSON.stringify({ files, ...extra })
        })
        .then(() => done(true))   // 성공: 버튼 복원
        .catch(() => done(false)); // 실패: 버튼 복원
    }
});

uploader.init(); // DOM 준비 후 호출js

createUploader(options?)

filePort의 유일한 진입점입니다. 옵션 객체를 받아 FilePort 인스턴스를 반환합니다. 

import { createUploader } from 'fileport';

const uploader = createUploader(options);
uploader.init();ts
반드시 DOM이 준비된 후 init()을 호출해야 합니다. (DOMContentLoaded, useEffect, 페이지 하단 <script> 등)

UploaderOptions

모든 옵션은 선택 사항입니다. 기본값만으로도 즉시 동작합니다.

옵션 타입 기본값 설명
uploadUrl string '/upload' 파일을 전송할 서버 엔드포인트 URL
maxFilesize number 10 파일 최대 크기 (MB 단위)
acceptedFiles string | null null 허용할 파일 확장자. 예: '.pdf,.docx,.xlsx'
showDocKind boolean true 문서 종류 열 표시 여부
simple boolean false true 시 체크박스·문서종류 없는 4열 간단 모드
docKindList DocKind[] [] 문서 종류 드롭다운 목록. { id, name }[]
submitBtnId string 'btnInsert' 등록 버튼 요소의 ID
elementConfig Partial<ElementConfig> 기본 ID 맵 DOM 요소 ID 재정의. 상세는 ElementConfig 참조
getExtra () => Record<string, unknown> () => ({}) 등록 시 함께 전송할 추가 폼 데이터를 반환하는 함수
getMessage (key: string) => string 한국어 메시지 UI 메시지 다국어 처리 함수
onSubmit (payload: SubmitPayload) => void () => {} 업로드 완료 후 서버 등록 로직을 구현하는 콜백

인스턴스 API

createUploader()가 반환하는 FilePort 인스턴스의 메서드와 프로퍼티입니다.

메서드

메서드설명
init() Dropzone, 이벤트 핸들러, 렌더러를 초기화합니다. DOM 준비 후 한 번 호출합니다.
startSubmit() 업로드 프로세스를 시작하거나 미업로드 파일이 없으면 onSubmit을 직접 호출합니다.
addFileInfo(data, type) 외부에서 파일 정보를 추가합니다.
type: 'scan' — 스캐너 등 외부 경로 파일
type: 'modify' — 기존 등록 파일 (수정 모드)
toggleDocKind(show) 문서 종류 열을 동적으로 표시/숨김 전환합니다.

프로퍼티

프로퍼티타입설명
fileManager FileManager 파일 상태 관리 객체.
.files — 현재 파일 배열
.updateStatus(id, status, time) — 상태 변경
.removeFile(id) — 파일 제거
myDropzone DropzoneInstance | null Dropzone 인스턴스에 직접 접근. 컴포넌트 언마운트 시 .destroy() 호출.
isProcessing boolean 현재 업로드 진행 중 여부.

SubmitPayload

onSubmit 콜백이 받는 페이로드 객체입니다.

interface SubmitPayload {
    files: SubmitFileItem[];            // 업로드된 파일 목록
    extra: Record<string, unknown>;    // getExtra() 반환값
    done:  (success?: boolean) => void; // 서버 처리 완료 후 호출
}ts
반드시 done()을 호출하세요. 서버 요청 성공·실패에 관계없이 done(true) 또는 done(false)를 호출해야 등록 버튼이 다시 활성화됩니다.

SubmitFileItem 구조

interface SubmitFileItem {

    isScanned?:   boolean;
    isOld?:       boolean;      // 수정 모드의 기존 파일

예제 — React

useRef와 useEffect를 사용해 컴포넌트 생명주기와 연동합니다.

import { useEffect, useRef } from 'react';
import { createUploader, type FilePort } from 'fileport';
import 'fileport/dist/filePort.css';

export default function FileUploader({ contentsId }: { contentsId: string }) {
    const uploaderRef = useRef<FilePort | null>(null);

    useEffect(() => {
        const uploader = createUploader({
            uploadUrl: '/api/upload',
            docKindList: [
                { id: 'CONTRACT', name: '계약서' },
                { id: 'REPORT',   name: '보고서' },
            ],
            getExtra: () => ({ contentsId }),
            onSubmit: async ({ files, extra, done }) => {
                try {
                    await fetch('/api/save', {
                        method: 'POST',
                        headers: { 'Content-Type': 'application/json' },
                        body: JSON.stringify({ files, ...extra }),
                    });
                    done(true);
                } catch {
                    done(false);
                }
            },
        });
        uploader.init();
        uploaderRef.current = uploader;

        // 언마운트 시 Dropzone 정리
        return () => {
            uploaderRef.current?.myDropzone?.destroy();
        };
    }, []);

    const handleSubmit = () => {
        uploaderRef.current?.startSubmit();
    };

    return (
        <div>
            <div id="tableWrapper" className="table-wrapper">
                <div className="table-scroll">
                    <table id="fileTable">
                        <thead>
                            <tr>
                                <th><input type="checkbox" id="checkAll" /></th>
                                <th>파일명</th>
                                <th>문서종류</th>
                                <th>크기</th>
                                <th>상태</th>
                                <th>작업</th>
                            </tr>
                        </thead>
                        <tbody id="fileList"></tbody>
                    </table>
                </div>
                <div className="table-footer">
                    <span id="footerSize"></span>
                    <div className="footer-progress">
                        <div className="progress-bar-wrap">
                            <div className="progress-bar-fill" id="progressFill"></div>
                        </div>
                        <span id="footerPercent"></span>
                    </div>
                </div>
            </div>
            <button id="btnFileAdd">파일 추가</button>
            <button id="btnInsert" onClick={handleSubmit}>등록</button>
        </div>
    );
}tsx

예제 — Spring Boot

서버에서 파일을 수신하는 Spring Boot 컨트롤러 예시입니다.

@RestController
@RequestMapping("/api")
public class FileController {

    // 1. 파일 바이너리 업로드 (Dropzone이 자동 호출)
    @PostMapping("/upload")
    public ResponseEntity<Map<String, Object>> upload(
            @RequestParam("file") MultipartFile file) throws IOException {

        String savedName = UUID.randomUUID() + "_" + file.getOriginalFilename();
        Path dest = Paths.get("/uploads/" + savedName);
        Files.copy(file.getInputStream(), dest);

        // filePort는 응답 JSON의 "id" 필드를 serverFileNm으로 사용합니다
        return ResponseEntity.ok(Map.of("id", savedName));
    }

    // 2. 메타 데이터 등록 (onSubmit 콜백에서 호출)
    @PostMapping("/contents/save")
    public ResponseEntity<Void> save(@RequestBody SaveRequest req) {
        // req.getFiles() → List<SubmitFileItem>
        // req.getContentsId(), req.getUserId() → extra 필드
        contentService.save(req);
        return ResponseEntity.ok().build();
    }
}java
응답 형식: /upload 엔드포인트는 반드시 { "id": "서버저장파일명" } 형태의 JSON을 반환해야 합니다. id 값이 SubmitFileItem.serverFileNm에 저장됩니다.

업로드 플로우

파일 종류와 상황에 따른 처리 흐름입니다.

신규 파일만 있는 경우

startSubmit() → Dropzone processQueue() → 파일별 /upload 전송 → queuecomplete → onSubmit() 호출

스캔 파일만 있는 경우

addFileInfo(data, 'scan') → startSubmit() → onSubmit() 직접 호출

혼합 (스캔 + 신규)

addFileInfo + 드래그&드롭 → startSubmit() → 신규 파일 업로드 → queuecomplete → onSubmit()

업로드 실패 처리

개별 파일 업로드가 실패하면 해당 파일은 목록에서 제거되고, 성공한 파일만 onSubmit의 files 배열에 포함됩니다. 등록 버튼은 done() 호출 전까지 비활성화 상태를 유지합니다.

스캔 파일 모드

스캐너 API, 로컬 앱 등 Dropzone 외부 경로로 파일을 추가할 때 사용합니다. addFileInfo(data, 'scan')으로 추가된 파일은 업로드 단계 없이 바로 onSubmit 페이로드에 포함됩니다. 

// 스캐너 API 응답 예시
const scannedFiles = [

];

uploader.addFileInfo(scannedFiles, 'scan');

// 이후 startSubmit() 호출 시 스캔 파일도 포함하여 onSubmit 실행

수정 모드

이미 등록된 파일 목록을 수정 화면에서 표시할 때 사용합니다. addFileInfo(data, 'modify')로 기존 파일을 로드하면 사용자가 파일을 추가·삭제하거나 문서 종류를 변경할 수 있습니다. onSubmit 페이로드의 isOld: true 플래그로 신규·기존 파일을 구분합니다. 

// 서버에서 기존 파일 목록 조회 후 로드
const existingFiles = await fetch('/api/files?contentsId=123').then(r => r.json());

uploader.addFileInfo(existingFiles, 'modify');
uploader.init();js
// onSubmit 콜백에서 신규/기존 파일 구분
onSubmit: ({ files, extra, done }) => {
    const newFiles = files.filter(f => !f.isOld);
    const oldFiles = files.filter(f =>  f.isOld);

    fetch('/api/contents/update', {
        method: 'PUT',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ newFiles, oldFiles, ...extra }),
    }).then(() => done(true)).catch(() => done(false));
